In order to submit a component to the design system, the component must go through a series of checks to ensure we consider things such as accessibility. All existing or live components that have not been checked across DVSA should be added to the backlog.

Do I need this new custom component?

Before designing any new components or patterns, always refer to the GDS design system for guidance on how you could solve a design problem. If you cannot find a solution from GDS, only then should a custom component be considered.

Quality gates

To ensure all new components match the standard that GDS have set, we have put together 'quality gates' as a way to test components, these are:

  • Does your component meet the WCAG 2 AA accessibility standards?
  • Has your component been user tested?
  • Has the markup of the component been reviewed?
  • Has the CSS and styling of the component been reviewed?

As the crossover between some quality gates are clear, it is important to ask each one individually. A component that has great feedback from user testing may not be accessible, therefore it's important to not overlook these steps.

Design system component flow

Each component should go through a similar flow to the graphic, being tested for 'quality gates'. Once each component has passed all quality gates, we can prepare them for the design system.
